Disk

24 May 2020, 08:54

磁盘结构

扇区 512 字节，是磁盘读写单位，block 由多个扇区组成，是软件创建的逻辑组，用来提高读写效率，通常 4 或 8K 字节 注意，OS 每次磁盘读写操作，总是访问至少一个 block

磁盘如何连接到计算机系统

磁盘控制器作用：

  • Control movement of disk heads
  • Select a sector on a track
  • Transfers data between disk surface and main memory
  • Disk controller has an internal buffer to store read/write data to/from disk
  • 1 disk controller can control n disks

硬盘结构

从上面看

  • Platters are 2-sided and magnetic
  • Platters rotates (about 10000 RPM)
  • All disk heads move at the same time (in or out)
  • Platter has circular tracks
  • Tracks are divided into sectors
  • Sector = the unit of write operation for a disk

However:Sector is too small to be efficient

  • Computer systems read/write a block (= multiple sectors) at once.
  • Gaps are non-magnetic and used to identify the start of a sector

cylinder

  • Cylinder = all tracks at the same distance from the center
  • Disk head does not need to move when accessing (read/write) data in the same cylinder
  • Block = a logical group (created by software) of sectors
  • A block is the (logical) unit of read/write performed by the Operating System

目录存储，有专门的位置

The directory is stored in a group of specially reserved blocks/tracks called: directory blocks/tracks

加速数据访问

通用方法：

  • Place sectors of blocks (= related) in same cylinder
  • Allocate blocks of the same file in same or nearby cylinders(Disk de-fragmentation)

硬件方面：

Use multiple smaller disks instead of one large disk

软件方面：

预取技术：OS 通常使用

The buffers in the operating system are organized as a circular buffer

User program does not need to use pre-fetching and uses 1 buffer to read a file

When the user program opens a file, the operating system will allocate N buffers and fill them

When the user program reads from the file, the operating system will transfer a buffer to the user buffer

The user program can process the data block in its buffer. At the same time, The operating system will read the next block into the emptied buffer

有效的调度算法

There are many disk requests at any one moment in time; Service policy: in which order do you process (= service) the pending requests，pending 请求才能调度

  1. First Come, First Serve

Advantage: Fairness !!!

Disadvantage: Inefficient !!!

Example:

  • Request 1: accesses data stored on track 1
  • Request 2: accesses data stored on track 1001
  • Request 3: accesses data stored on track 2
  • Request 4: accesses data stored on track 1002 and so on…

The disk head will move 1000 + 999 + 1000 = 2999 tracks !!!!!Because seek time is very long for a disk, moving 2999 tracks is very time consuming

  1. elevator

The disk head sweeps in-and-out (like an elevator), When the disk head is on a cylinder k: Disk will service all requests for that cylinder before moving to the next cylinder

上面的例子使用电梯算法调度：

  • Request 1: accesses data stored on track 1 — processed first
  • Request 3: accesses data stored on track 2 !!!! (disk head passes by track 2 next and service request !!!)
  • Request 2: accesses data stored on track 1001
  • Request 4: accesses data stored on track 1002

Number of tracks traversed = 1001 !!!

Advantage: Efficiency

Disadvantage: Unfair !!!
