Shared Memory Machine Model

26 Mar 2020, 11:51

介绍 3 种不同结构的共享内存机器模型，共享内存意思是每个 CPU 都可以访问所有内存空间，但缓存是每个 CPU 独有的，无法相互访问

Dance Hall Architecture

SMP Architecture

Symmetric Multiprocessor，对称是因为每个 cpu 的访存时间相同，

Distributor Shared Memory Architecture

每个 CPU 有一个关联的内存，每个 CPU 都可以通过系统总线访问到其他 CPU 关联的内存，但是访问自己的内存比较快

Shared Memory And Caches

这里以 SMP 架构为例进行讲解。

单 CPU 架构下，CPU 访问内存时，首先访问 cache，如果 cache 命中，则直接访问，否则访问内存，同时将结果缓存在 cache 中；多 CPU 架构下 cache 的作用与单 CPU 一样，但是有个问题就是 cache 是 CPU 私有的，memory 是 CPU 共享的，假设内存有个热点变量 var 值为 y，缓存在所有 CPU 的 cache 中，当 cpu1 对私有 cache 中的 var 修改时，假设修改后的值为 x，那么所有 CPU 的 cache 中的 var 现在都应该是 x，这点应该由谁来保证呢？

例子

假设共享的内存变量 a 和 b 初始值为 0，c 和 d 也是共享的内存变量两个 CPU 执行各自的代码如下：

processor P1 processor P2
a = a+1; d = b;
b = b+1; c = a;

d 和 c 可能的结果是多少？

  • c = d = 0
  • c = d = 1
  • c = 1, d = 0
  • c = 0, d = 1

单 CPU 时候，指令按照程序指定的顺序执行；多 CPU 时候，单个 CPU 上的程序指令仍然按照程序指定的顺序执行，但是多个 CPU 上程序执行的顺序不确定；

当 P2 上的代码在 P1 之前执行就会出现第一种结果

当 P1 上的代码在 P2 之前执行就会出现第二种结果

当 P2 上的代码在 P1 代码之间执行就会出现第三种结果

当消息传递是无序的情况下，就会出现第四种结果（回想上图中各个处理器通过 interconnection network 相连，a = a + 1 消息如果在 b = b + 1 消息之后到达 P2，就会发生这种情况）

第四种结果很奇怪，所以有内存一致性模型来防止这种情况发生

Memory Consistency Model

processor P1 processor P2
r(a) w(a)
w(b) r(b)
w(c) w(c)
r(b)

a,b,c,d 为共享内存变量（以后不再强调共享，简称内存变量），r 和 w 表示读和写内存访问，P1 和 P2 独立访存

sequential consistency 一致性模型，如上图所示，是由绿色箭头所表示的 program order 和红色箭头所表示的代码之间的 arbitrary interleaving 组成；就像扑克牌洗牌一样，一副扑克牌分成两部分，每一部分是有序的，但是洗完牌后，两部分交叉；两部分各自的顺序在最后的牌中不变，但两部分之间的顺序在最后的牌中是不确定的，交叉的

在 sequential consistency 一致性模型下，上面例子中的第四种结果就不会出现了，同时也不会让程序员费解

内存一致性模型是呈现给程序员的模型，而缓存一致性是在存在私有缓存的情况下，系统如何实现该一致性模型；有可能是系统软件实现，当用户修改内存数据时，系统软件实现缓存一致性；也可以由硬件实现，缓存一致性；如果由硬件实现缓存一致性，有两种可能情况：

1.write invalidate schema

想法很简单，就是当内存数据缓存在各个 CPU 的缓存中时，如果其中一个 CPU 修改该内存位置的数据了，那么硬件会通过系统总线广播该消息，此时所有缓存在其他 CPU 中的该位置的数据标记被为 invalidate（如果缓存在本地缓存，没有则忽略）

2.write update

这种情况下，如果其中一个 CPU 修改该内存位置的数据，广播的消息是该内存位置被修改了，新的值为多少，这样所有其他 CPU 中该位置的数据将被更新为新的数据

不论上面哪种模式，都是需要在系统总线上广播消息，来通知其他 CPU 的 cache，这是有开销的，而系统设计目标之一就是尽量减少开销，而且可以看到，随着 CPU 的增多，这种开销也在增加

所以如何获取高性能？在线程间尽可能减少共享内存，多核计算机可以非常好的进行扩展当不存在共享内存的时候；

多 CPU 共享内存，对同步（synchronization）算法，进程间通信（IPC），和调度算法（schedule）都产生影响，请记住这一点
