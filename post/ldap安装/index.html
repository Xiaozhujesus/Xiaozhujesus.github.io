Ldap安装

12 Mar 2020, 11:02

安装前环境准备

在安装OPENLDAP之前需要关闭防火墙

[wz]# systemctl stop firewalld.service
[wz]# systemctl disable firewalld.service

OPENLDAP-servers安装及配置

安装

[wz]# yum install -y openldap-servers

配置

  1. 配置管理员密码
[wz]# slappasswd -s passwd
{SSHA}2e1yDU9g9VSc8UArkfpi5KsjT9mSZxCN
  1. 修改配置文件

olcDbDirectory：指定ldap数据库的路径；

olcSuffix：指定管理员管理的目录树根域；

olcRootDN：指定管理员dn；

olcRootPW：该字段需要自行添加，用于指定管理员密码，将上一步生成的加密字符串添加在这一行；

[wz]# vim /etc/openldap/slapd.d/cn\=config/olcDatabase\=\{2\}hdb.ldif

# AUTO-GENERATED FILE - DO NOT EDIT!! Use ldapmodify.
# CRC32 1763c834
dn: olcDatabase={2}hdb
objectClass: olcDatabaseConfig
objectClass: olcHdbConfig
olcDatabase: {2}hdb
olcDbDirectory: /var/lib/ldap
olcSuffix: dc=com
olcRootDN: cn=Manager,dc=com
olcRootPW: {SSHA}2e1yDU9g9VSc8UArkfpi5KsjT9mSZxCN
olcDbIndex: objectClass eq,pres
olcDbIndex: ou,cn,mail,surname,givenname eq,pres,sub
structuralObjectClass: olcHdbConfig
entryUUID: b5baf318-0bee-1039-9713-29cc6215f8f1
creatorsName: cn=config
createTimestamp: 20190516062222Z
entryCSN: 20190516062222.655732Z#000000#000#000000
modifiersName: cn=config
modifyTimestamp: 20190516062222Z
  1. 配置监控

将olcAccess字段中read by dn.base="cn=xxxxxxxx"修改为read by dn.base="cn=Manager,dc=com”

[wz]# vim /etc/openldap/slapd.d/cn\=config/olcDatabase\=\{1\}monitor.ldif
 
# AUTO-GENERATED FILE - DO NOT EDIT!! Use ldapmodify.
# CRC32 36507a81
dn: olcDatabase={1}monitor
objectClass: olcDatabaseConfig
olcDatabase: {1}monitor
olcAccess: {0}to * by dn.base="gidNumber=0+uidNumber=0,cn=peercred,cn=extern
 al,cn=auth" read by dn.base="cn=Manager,dc=com" read by * none
structuralObjectClass: olcDatabaseConfig
entryUUID: b5bae652-0bee-1039-9712-29cc6215f8f1
creatorsName: cn=config
createTimestamp: 20190516062222Z
entryCSN: 20190516062222.655404Z#000000#000#000000
modifiersName: cn=config
modifyTimestamp: 20190516062222Z
  1. 配置数据库

OPENLDAP默认使用Berkeley DB，数据库的路径由配置文件olcDatabase={2}hdb.ldif中olcDbDirectory字段指明

[wz]# cp /usr/share/openldap-servers/DB_CONFIG.example /var/lib/ldap/DB_CONFIG
[wz]# chown ldap:ldap -R /var/lib/ldap
[wz]# chmod 700 -R /var/lib/ldap
  1. 导入schema

将一些基本的schema导入OPENLDAP服务器

ls /etc/openldap/schema/*.ldif | xargs -I {} sudo ldapadd -Y EXTERNAL -H ldapi:/// -f {}
  1. 配置日志

默认情况下，OPENLDAP日志处于停用状态，可通过如下步骤配置并启用日志：

创建日志配置LDIF文件

[wz]# vim /root/loglevel.ldif

dn: cn=config
changetype: modify
replace: olcLogLevel
olcLogLevel: stats

将日志配置信息导入OPENLDAP服务器并重启服务

[wz]# ldapmodify -Y EXTERNAL -H ldapi:/// -f /root/loglevel.ldif
[wz]# systemctl restart slapd.service

修改rsyslog.conf配置文件并重启服务

[wz]# vim /etc/rsyslog.conf
local4.*                                                /var/log/slapd.log
[wz]]# systemctl restart rsyslog
  1. 测试配置
[wz]]# slaptest -u
5d09ed7b ldif_read_file: checksum error on "/etc/openldap/slapd.d/cn=config/olcDatabase={1}monitor.ldif"
5d09ed7b ldif_read_file: checksum error on "/etc/openldap/slapd.d/cn=config/olcDatabase={2}hdb.ldif"
config file testing succeeded
  1. 启动并设置开机启动
[wz]# systemctl start slapd.service
[wz]# systemctl enable slapd.service

OPENLDAP-clients安装及配置

安装OPENLDAP-clients

[wz]# yum install -y openldap-clients

配置OPENLDAP-clients标定OPENLDAP-servers

BASE：指定OPENLDAP目录树的根域

URI：指定OPENLDAP服务器地址

[wz]# vim /etc/openldap/ldap.conf
BASE    dc=mlhadoop,dc=com
URI     ldap://hostname:389
