Linux File

21 May 2020, 08:00

大多数文件 IO 用到 5 个函数，这些函数是系统调用而不是库函数，会陷入内核

  • open
  • read
  • write
  • lseek
  • close

open

必选项如下：

  • O_RDONLY
  • O_WRONLY
  • O_RDWR
  • O_EXEC
  • O_SEARCH

可选项如下：

  • O_APPEND
  • O_CLOEXEC
  • O_CREAT
  • O_DIRECTORY
  • O_EXCL
  • O_NOFOLLOW
  • O_NOBLOCK
  • O_TRUNC
  • O_SYNC
  • O_DSYNC
  • O_RSYNC

这里主要说下可选项的后三个

O_SYNC

每次 write 等待物理 IO 操作完成，包括由该 write 引起的文件属性更新所需要的 IO

O_DSYNC

每次 write 要等待物理 IO 操作完成，但是如果该写操作并不影响读取刚写入的数据，则不需要等待文件属性被更新

O_DSYNC 与 O_SYNC 有微妙的区别，仅当文件属性需要更新以反映文件数据变化时候，例如更新文件大小以反应文件中有更多数据，就可以看出二者的区别；设置 O_SYNC 后文件数据和属性总是同步更新，而 O_DSYNC 选项，当使用 write 重写文件现有部分内容时候，文件时间属性不会同步更新，但是会更新，只是 write 返回时候只是保证数据被更新了，而文件元数据则会在稍后某个时间更新

O_RSYNC

使得每个以文件描述符为参数的 read 操作等待，直至所有对文件同一部分挂起的写操作都完成

Solaris 支持所有这 3 个选项，而 Linux 处理 O_RSYNC 与处理 O_SYNC 相同

close

关闭一个打开的文件，进程终止时候，内核会自动关闭它所有打开的文件

lseek

每个打开的文件都有一个与其关联的当前文件偏移量，通常，读写操作都从当前文件偏移量开始，并使偏移量增加所读写的字节数；默认情况，除非指定 O_APPEND 选项，否则当 open 一个文件时候，文件偏移量被设置为 0

文件偏移量可以大于文件当前长度，这种情况下，对该文件的下一次写操作会在当前偏移量基础上加上写入字节数，文件中形成空洞，文件中没有写过的字节，read 时候会得到 0 值，文件空洞在磁盘上并不占用磁盘空间，对于新写的数据需要分配新的磁盘块，空洞不需要磁盘块

$ ls -ls hole nohole
8  -rw-r--r-- sar 16394 Nov 25 01:01 hole
20 -rw-r--r-- sar 16394 Nov 25 01:01 nohole

-s 选项可以查看磁盘块占用，可以看到有空洞的文件只占用了 8 个块

read

读操作从文件当前偏移量开始，成功返回之前，该偏移量将增加实际读到的字节数

write

写操作从文件当前偏移量开始，如果打开文件时候指定了 O_APPEND 选项，则每次写之前将偏移量设置在文件当前结尾处
